# Markdown Best Practices Summary

## Quick Reference

| Category | Best Practice |
|----------|---------------|
| Headings | Single H1, logical hierarchy, don't skip levels |
| Spacing | Blank lines before/after headings and blocks |
| Lists | Consistent markers (`-` or `*`), proper indentation |
| Code | Specify language in fenced blocks |
| Links | Descriptive text, not "click here" |
| Images | Always include alt text |

## The 6 Golden Rules

1. **One H1 per document** - Use it as your title
2. **Blank lines matter** - Add them around headings, lists, and code blocks
3. **Be consistent** - Pick one style for lists and stick to it
4. **Specify languages** - Always add language to code fences for highlighting
5. **Descriptive links** - "See the [installation guide]()" not "[click here]()"
6. **Alt text for images** - `![Description](image.png)` for accessibility

## Common Mistakes

```markdown
# Bad: Skipping heading levels
# Title
### Subsection (skipped H2!)

# Good: Proper hierarchy
# Title
## Section
### Subsection
```

```markdown
# Bad: No blank lines
# Heading
Some text right after
- list item

# Good: Proper spacing
# Heading

Some text with spacing.

- list item
```

## Template

```markdown
# Document Title

Brief introduction paragraph.

## Section 1

Content here.

### Subsection 1.1

More details.

## Section 2

- List item 1
- List item 2

## Code Example

\`\`\`python
def hello():
    print("Hello, World!")
\`\`\`

## References

- [Link 1](url) - Description
- [Link 2](url) - Description
```
